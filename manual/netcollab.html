◆Network canvas [BETA]

Network canvas allows you to do real-time collaboration with multiple users.

Network canvas is currently in beta phase.
Some features may not work as expected.
Save your work often.


◆macOS:
Network canvas features are currently not available on macOS.

◆Start a network canvas session server

To start a network canvas session, open an editor workspace and select File > Start network canvas server on the navigation bar.

You will have an option to choose your username, user color and the port to use.

Click Confirm to start the server.

The server will be hosted on all available network interfaces.
For a server to be accessible, one of the following conditions must be met:
  • The server is on the same local network as the client (connected to the same router or network switch)
  • The server is on the same virtual network as the client (like Hamachi or ZeroTier)
  • The server is properly port-forwarded for TCP connections


◆Windows:
When hosting a server for the first time, you will receive a Windows Firewall popup to allow or deny voidsprite to use network sockets.
To host or connect to a server, you must allow this, ideally for both private and public connections.
If you deny, you must manually allow voidsprite in the Windows Firewall settings.

◆Connect to a network canvas session

From the voidsprite Launchpad, select File > Connect to network canvas on the navigation bar.

Enter the IP address/hostname and port of the server you want to connect to.
Enter your username and set your user color, then click Confirm to connect.

◆Hostnames and IPv6 addresses

For easier connection, you can use hostnames instead of IP addresses.
On Windows local networks, you can use the name of the computer, such as DESKTOP-1234567.

You may also use IPv6 addresses.
