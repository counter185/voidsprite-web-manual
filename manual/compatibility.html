◆Compatibility list

This page lists the format compatibility of voidsprite.

◆Image formats

◆Image import/export

Name Extension Import Export
  • OpenRaster: advanced features from painting programs like layer blend modes, etc. are not available here.
    Exporting the thumbnail will not work until color quantization is implemented.
  • Pixel Studio sessions: a 1:1 read of this format requires accurately implementing every tool from that program.
    For the best experience, wipe the undo history before attempting an import (Functions -> Resize canvas -> Resize -> Yes).
    Animation features are not supported.
  • VTF: no mipmaps are imported or exported.
    Formats: I8, IA88, A8, RGB565,BGR888,RGB888, BGRA8888, RGBA8888, ARGB8888, ABGR8888, DXT1, DXT3, DXT5
  • DIBv5: JPEG, RLE4 and RLE8 subformats currently not supported.
  • Aseprite: experimental. Only frame 1 is loaded. Layer blend modes and tilemaps are not available here.

◆Image import-only

Name Extension Import Export
  • Wii TPL: only I4, RGB5A3, RGBA32.
  • AETEX: GXT and Switch ASTC subformats not implemented
  • DDS: only BC1, BC2, BC3 and BGRA8888

◆Image export-only

Name Extension Import Export
  • Java BufferedImage: only works for small images. Larger images go beyond the function size limit.
