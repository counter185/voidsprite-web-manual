◆Application data directory

The application data directory stores the configuration and user content.
To open it in the system's file manager, open File > Preferences and go to Misc. > Open app data directory...

◆Platform-specific location

The location of the application data directory depends on the platform:

  • Windows: %APPDATA%/voidsprite
  • macOS: ~/Library/Application Support/voidsprite
  • Linux: ~/.var/app/com.github.counter185.voidsprite/config/voidsprite
  • Android: /sdcard/Android/data/pl.cntrpl.voidsprite/files

◆What's in the application data directory

The application data directory contains the following files and directories:

  • 9segmentpatterns/Stores 9-segment patterns.
  • autosaves/Contains recovery autosaves. Any .voidsn file here will appear on the Recovery autosaves list.
  • palettes/Putting compatible color palette files here will make them usable from the Palettes list in the Editor.
  • patterns/Putting images in Portable Bitmap .pbm or .xbm format here will make them usable from the pattern list in the Editor.
  • templates/Putting images here will make them usable from the Templates list on the Launchpad.
  • plugins/Any native plugins in this folder will be loaded on launch.
  • visualconfigs/Stores custom visual configurations.
  • config.txtStores the configuration. Deleting this will reset all settings and keybinds to their defaults.
  • voidsprite_log.txtLog output created when voidsprite is ran. Contains information useful for debugging, so attach this file when reporting issues.
